# Deployment Guide for Hostinger (GitHub Integration)

To make your web app live on your domain (`www.bookmyvanue.in`) using the code from GitHub, follow these steps in your Hostinger Panel:

## 1. Setup Node.js Application
1. Go to **Websites** -> **Manage** -> **Node.js**.
2. Click **Create Application**.
3. **Application Name:** `bookmyvanue`
4. **Application URL:** `www.bookmyvanue.in`
5. **Node.js Version:** Select **20.x** or **18.x**.
6. **Application Entry Point:** `server.js` (This is the file generated by the build).
7. **Application Root:** `/` (or the folder where your code is).

## 2. Connect GitHub Repository
1. In the Node.js App settings, find the **GitHub** section.
2. Connect your repository.
3. Set the branch (usually `main`).
4. **IMPORTANT:** Hostinger will pull the code, but it needs to build it. I have added a `postinstall` script in `package.json` that will automatically run the build (`npm run build`) when Hostinger runs `npm install`.

## 3. Configure Environment Variables
Your app needs Supabase and Razorpay keys to work. Hostinger does NOT read `.env` files automatically in the Node.js setup UI.
1. In the Hostinger Node.js App settings, find the **Environment Variables** section.
2. Add the following keys (copy values from your local `.env` or Supabase/Razorpay dashboards):
   - `NODE_ENV` = `production`
   - `VITE_SUPABASE_URL` = `your_supabase_url`
   - `VITE_SUPABASE_ANON_KEY` = `your_supabase_anon_key`
   - `VITE_RAZORPAY_KEY_ID` = `your_razorpay_key_id`
   - `RAZORPAY_KEY_SECRET` = `your_razorpay_key_secret`
   - `PORT` = `3000` (Hostinger will proxy this to your domain).

## 4. Trigger Deployment
1. Click **Install Dependencies** (or **NPM Install**) in the Hostinger panel.
2. This will trigger the `postinstall` script I added, which runs `npm run build`.
3. Once finished, click **Start Application**.

## Troubleshooting 503 Error
- If you see "Server is temporarily busy", it means the Node.js process crashed or is still starting.
- Check the **Log Viewer** in the Hostinger Node.js panel.
- Look for messages starting with `>`. I have added detailed logging to help you see if a variable is missing or if the `dist` folder is not found.

**Note:** Since you are using Supabase, you do NOT need to use Hostinger's phpMyAdmin. All your data is safely stored in Supabase.
